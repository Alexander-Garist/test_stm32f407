git шпаргалки

1. Для создания ветки feature_... алгоритм действий такой:
	Клонировать проект себе
	В каталоге проекта где находится .git открыть git bash и выполнить действия:
		git checkout -b <name>		Создать новую ветку локально
		git status					Просмотреть что отличается от основной ветки, из которой был сделан клон
		git add 					Добавить в коммит нужные файлы		
		git commit (без аргументов) Откроется текстовый редактор, нажать i, ввести заголовок и описание коммита, затем Esc и :wq для выхода из vim
		git push -u origin <name>	В удаленном репозитории появится ветка с таким же именем <name>, связанная с локальной и произойдет пуш в удаленный репозиторий

В результате будет 1 ветка для 1 фичи и сразу видно что было изменено в рамках этой 1 фичи, так будет намного быстрее и проще мержить.

2. Во время работы в одной ветке проекта нужно переключиться на другую, при этом сделать сейчас коммит чтобы сохранить изменения нельзя (что-то не доделано/не работает/не готово => такое коммитить нельзя)
git stash			Откат локальной ветки до последнего коммита, все текущие незакоммиченные изменения сохраняются
git checkout		Переход в другую ветку, выполнение каких-то действий
...
git checkout		Возврат в исходную ветку
git stash pop		Возврат всех незакоммиченных изменений, которые были до вызова git stash




git add -A
git rm
git mv

git checkout <имя ветки> -- <файл>		позволяет втянуть в текущую ветку конкретный файл из другой ветки
git commit --amend --no-edit			позволяет изменить предыдущий коммит без изменения названия и комментария
git push -f 							Отправить изменения в существующий и уже запушенный ранее коммит

git branch							Показать только локальные ветки 
git branch -d <name>				Удалить локальную ветку
git push origin --delete <name>		Удалить ветку из удаленного репозитория



/** Слияние 2 веток, для этого перейти в ветку, в которую нужно что-то влить */
git fetch 					Получить данные об изменениях в удаленной ветке
git pull origin <name>		Обновить локальную ветку по удаленной
git merge <local_branch>	Влить <local_branch> в текущую ветку
git push					Отправить коммит с мерджем на удаленный репозиторий
